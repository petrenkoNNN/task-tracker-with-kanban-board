# Пользовательские истории (Бэклог MVP)
**Проект:** Веб-трекер задач с канбан-доской
 
**Версия:** 1.0 (Первый спринт)
 
## Эпик 1: Управление жизненным циклом задачи (CRUD)
*Основные действия с карточками на доске.*
 
### 1.1. Создание новой задачи
 
Как пользователь, я хочу иметь форму для ввода названия и описания задачи, чтобы фиксировать новые дела в списке "Нужно сделать".
 
- **Критерии приемки (AC):**
 
    - Поле "Название" обязательно для заполнения.
 
    - Поле "Описание" — опционально.
 
    - После нажатия "Сохранить" задача появляется в первой колонке (todo).
 
    - Форма очищается после успешного создания.
 
### 1.2. Просмотр списка задач
 
Как пользователь, я хочу видеть все свои задачи распределенными по 3 колонкам, чтобы понимать общую картину проекта.
 
- **Критерии приемки (AC):**
 
    - Колонки: "Нужно сделать", "В работе", "Готово".
 
    - Карточка отображает заголовок (полностью) и кнопки действий.
 
### 1.3. Редактирование задачи
 
Как пользователь, я хочу изменять текст уже созданной задачи, чтобы актуализировать информацию без удаления карточки.
 
- **Критерии приемки (AC):**
 
    - При клике на "Редактировать" открывается форма с предзаполненными данными.
 
    - Изменения сохраняются в базе данных/LocalStorage.
 
### 1.4. Удаление задачи
 
Как пользователь, я хочу иметь возможность безвозвратно удалить задачу, чтобы она не занимала место на доске.
 
- **Критерии приемки (AC):**
 
    - Удаление происходит по нажатию на кнопку "Удалить".
 
    - Опционально: Система запрашивает подтверждение удаления.
 
## Эпик 2: Работа с канбан-механикой
*Логика перемещения и статусов.*
 
### 2.1. Смена статуса (Перемещение)
 
Как пользователь, я хочу передвигать задачу в следующую или предыдущую колонку, чтобы отражать прогресс выполнения.
 
- **Критерии приемки (AC):**
 
    - Перемещение кнопками "Влево"/"Вправо" или Drag-and-Drop.
 
    - Задача в крайней левой колонке не может быть перемещена еще левее.
 
    - Статус задачи обновляется в модели данных мгновенно.
 
### 2.2. Пустые состояния колонок
 
Как пользователь, я хочу видеть индикатор, если в колонке нет задач, чтобы понимать, что интерфейс не сломался, а просто пуст.
 
- **Критерии приемки (AC):**
 
    - Отображение текста "Здесь пока пусто", если массив задач для данного статуса пуст.
 
## Эпик 3: Технические требования (System Stories)
*Требования к архитектуре и качеству.*
 
### 3.1. Сохранение состояния (Persistence)
Как разработчик, я хочу интегрировать LocalStorage, чтобы пользователь не терял данные при обновлении страницы (F5).
 
### 3.2. Валидация данных
Как аналитик, я хочу, чтобы система блокировала сохранение пустых заголовков, чтобы поддерживать чистоту данных в БД.
